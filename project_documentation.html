Multimodal RAG Incident Analyzer

An AI-powered DevOps incident diagnosis system that analyzes system logs, Kubernetes events, and dashboard screenshots using a Retrieval-Augmented Generation (RAG) pipeline.

Technology Stack

Python 3.11 FastAPI Pydantic
SentenceTransformers OpenAI CLIP LLMs (OpenAI/Fallback)
Qdrant Vector DB Docker HTML/CSS/JS UI

System Architecture

The system orchestrates a complex multimodal pipeline to diagnose production incidents:

  1. Input Parsing: Accepts syslog, Kubernetes events, generic text, and file uploads. Extracts key error signals and timestamps.
  2. Text Embedding: Uses all-MiniLM-L6-v2 (SentenceTransformers) to convert log context and error signals into 384-dimensional vector embeddings.
  3. Image Encoding: Processes uploaded dashboard screenshots (e.g., Grafana/Prometheus) using OpenAI's CLIP (ViT-B/32) model to extract visual context natively.
  4. Retrieval Layer: Queries the locally-hosted Qdrant vector database to fetch relevant runbooks, architecture docs, and past troubleshooting records.
  5. AI Reasoning: Constructs a comprehensive prompt containing log traces, image descriptions, and retrieved context. Sends it to an LLM (or a highly capable local heuristic engine fallback) to synthesize a structured Incident Report.

API Reference

Endpoint Method Description
/api/analyze POST Analyzes text logs or image files and returns a JSON IncidentReport.
/api/knowledge/ingest POST Embeds and stores a new runbook doc in the Qdrant DB.
/api/knowledge/documents GET Retrieves metadata for all stored context documents.
/api/knowledge/documents/{id} DELETE Removes a document from the vector store.
/health GET System diagnostics (DB connection, models loaded).

Deployment & Quick Start

Ensure Docker is running, then execute the following to start the Vector DB:

docker-compose up -d qdrant

Set up the Python environment and run the backend:

python -m venv venv
# Windows: .\venv\Scripts\activate
# Mac/Linux: source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000

Open http://localhost:8000 in your browser to access the graphical UI.
